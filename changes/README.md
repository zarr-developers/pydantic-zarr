# Changelog Entries

This directory contains changelog entries for the pydantic-zarr project.

## Adding a changelog entry

When making a pull request, you should add a changelog entry to this directory.
The entry should be in a file named `xxxx.<type>.md` where:

- `xxxx` is the pull request number
- `<type>` is one of: `feature`, `bugfix`, `doc`, `removal`, or `misc`

The file should contain a short description of what you have changed, and how it impacts users of `pydantic-zarr`.

## Fragment types

- `feature` - for new features
- `bugfix` - for bug fixes
- `doc` - for documentation changes
- `removal` - for removals
- `misc` - for miscellaneous changes that don't fit other categories
